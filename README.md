# AI 일본 문화 체험 실습 대시보드

관광일본어과 1반의 35분 AI 교류수업을 위해 만든 정적 수업 대시보드이자 프롬프트 빌더입니다. 일본 신사 문화의 세 가지 소재를 살펴보고, 다섯 가지 시각 테마 중 하나를 선택해 외부 생성형 AI에 전달할 웹앱 제작 프롬프트를 만들 수 있습니다.

## 현재 페이지와 생성 대상 앱의 차이

이 저장소의 `index.html`은 생성형 AI 모델이나 API를 호출하지 않습니다. 현재 페이지가 직접 제공하는 기능은 수업 안내, 문화 설명, 테마별 프롬프트 작성과 복사입니다.

프롬프트에 적힌 다음 기능은 **현재 페이지에 구현된 기능이 아니라**, 사용자가 복사한 프롬프트를 Gemini Canvas 등의 외부 도구에 입력해 별도로 만들도록 요구하는 목표 앱의 사양입니다.

- 버튼 순서에 따른 참배 예절 체험
- 무작위 오미쿠지 결과
- 이름과 소원을 입력하는 에마 목록

외부 AI가 생성한 앱의 동작, 정확성, 안전성은 이 저장소에서 보장하거나 검증하지 않습니다.

## 주요 기능

- Chart.js 도넛 차트로 표시하는 35분 수업 구성
  - 인트로 5분
  - AI 접속·시연 10분
  - 앱 생성 12분
  - 수정·공유 8분
- 일본 신사 문화 소재 카드 3개
  - `二礼二拍手一礼`
  - `おみくじ`
  - `絵馬`
- 생성 프롬프트의 분위기를 바꾸는 테마 5개
  - 교토의 봄
  - 단풍 신사
  - 대나무 숲
  - 붉은 도리이
  - 겨울 신사
- 현재 테마가 반영된 전체 프롬프트 복사
- 디자인, 일본어, 오류 수정의 고정 조언 3개 선택 및 복사
- 화면 크기에 따라 재배치되는 단일 페이지 UI

`AI Miko Guide` 영역은 미리 정의된 세 종류의 문구를 전환하는 인터페이스입니다. 사용자의 질문을 받거나 AI가 답변을 생성하는 채팅 기능은 아닙니다.

## 기술 구성

- HTML
- 인라인 CSS
- Vanilla JavaScript
- Tailwind CSS 브라우저 CDN
- Chart.js CDN
- Google Fonts CDN

빌드 도구, 패키지 관리자, 서버 애플리케이션은 사용하지 않습니다.

## 프로젝트 구성

```text
.
├── index.html   # 화면, 스타일, 차트와 상호작용 로직
└── README.md
```

모든 화면과 동작은 `index.html` 한 파일에 들어 있습니다.

## 실행 방법

### 파일 직접 열기

`index.html`을 브라우저로 열면 별도 설치 없이 페이지를 볼 수 있습니다. 단, Tailwind CSS, Chart.js와 Google Fonts를 외부에서 불러오므로 인터넷 연결이 필요합니다.

`file://`로 직접 연 페이지에서는 브라우저 정책에 따라 Clipboard API가 제공되지 않아 복사 버튼이 실패할 수 있습니다. 현재 코드는 복사 실패를 처리하는 대체 동작이나 오류 메시지가 없습니다.

### 로컬 정적 서버

Python 3가 설치되어 있다면 저장소 루트에서 정적 HTTP 서버를 실행할 수 있습니다.

```bash
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000`에 접속합니다. Clipboard API는 보안 컨텍스트에서만 사용할 수 있으므로, 원격 게시 시에는 HTTPS가 필요합니다. 브라우저 권한과 정책에 따라 localhost에서도 복사가 제한될 수 있습니다.

## 사용 방법

1. 수업 시간표와 세 가지 문화 소재를 확인합니다.
2. 프롬프트 빌더에서 원하는 테마를 선택합니다.
3. `전체 프롬프트 복사하기` 버튼으로 현재 프롬프트를 복사합니다.
4. 사용자가 선택한 외부 생성형 AI 도구에 직접 붙여넣어 별도 앱을 생성합니다.
5. 디자인, 일본어 또는 오류 수정 조언을 선택해 추가 프롬프트로 활용합니다.
6. 생성된 결과의 일본어, 문화 설명과 실제 기능을 사람이 다시 검토합니다.

이 과정에서 현재 페이지는 외부 AI에 데이터를 자동 전송하지 않습니다.

## 수업 목적

이 프로젝트는 코딩 문법 자체를 학습하기보다, 일본어와 일본 문화에 대한 전공 지식을 AI를 활용해 디지털 콘텐츠로 구체화하는 경험을 제공하는 데 목적이 있습니다.

학생은 AI가 생성한 일본어와 문화 설명을 그대로 받아들이지 않고, 전공 지식을 바탕으로 표현과 내용의 정확성을 확인하고 수정합니다.

## 데이터, API 및 환경 변수

- AI API 및 백엔드 API 호출 없음
- API 키와 환경 변수 없음
- 로그인, 인증과 권한 기능 없음
- 입력 폼과 사용자 데이터 수집 없음
- 데이터베이스, 쿠키와 로컬 스토리지 없음
- 선택한 테마와 조언은 현재 브라우저 메모리에만 반영되며 새로고침 시 초기화

CDN 자원을 내려받기 위한 브라우저 요청은 각 외부 CDN·글꼴 제공자에 전송됩니다.

## GitHub Pages 게시

이 저장소는 루트에 `index.html`이 있어 정적 호스팅 후보로 사용할 수 있습니다. GitHub Pages로 게시하려면 저장소 설정에서 배포 소스를 선택하고 브랜치를 `main`, 폴더를 `/(root)`로 지정합니다. `/root`라는 실제 하위 디렉터리를 선택하는 의미가 아닙니다.

현재 저장소에는 Pages 배포 워크플로, 사용자 지정 도메인 설정 또는 배포 결과를 검증하는 테스트가 없습니다. 실제 게시 여부와 공개 URL은 이 README에서 확인하지 않았습니다.

## 테스트 및 검증

자동화된 테스트와 빌드 구성이 없습니다. 변경 후에는 최소한 다음 항목을 브라우저에서 수동 확인해야 합니다.

- 35분 차트가 표시되고 합계가 35분인지
- 5개 테마 버튼이 프롬프트의 테마 문구를 변경하는지
- 전체 프롬프트 복사가 현재 테마를 포함하는지
- 디자인·일본어·오류 수정 버튼이 각각 다른 조언을 표시하는지
- 조언 복사와 복사 완료 토스트가 동작하는지
- 모바일과 데스크톱 화면에서 내용이 겹치지 않는지
- CDN이 차단되거나 오프라인일 때 페이지가 어떻게 보이는지

이 README 정비에서는 실제 Pages 배포와 외부 생성형 AI의 결과를 검증하지 않았습니다.

## 문화·언어 콘텐츠 검토

페이지와 생성 프롬프트에는 일본 신사 참배, 오미쿠지, 에마 및 일본어 표현에 관한 교육용 설명이 포함되어 있습니다. 코드만으로 문화적·언어적 정확성이나 모든 신사에 대한 일반화 가능성을 검증할 수는 없습니다.

수업 또는 공개 전 관광일본어·일본문화 담당자가 다음을 수동 검토해야 합니다.

- `二礼二拍手一礼` 설명과 적용 범위
- 오미쿠지 종류, 순서와 표현
- `絵馬` 관련 표현과 체험 설명
- 한자, 히라가나 읽기와 한국어 의미
- 고정된 문화 이미지를 강화하거나 종교적 맥락을 왜곡하는 표현 여부
- 외부 AI가 새로 만든 일본어·문화 설명의 정확성

현재 프롬프트의 원칙대로 일본어 발음을 한글로 옮기지 않고, 한자 읽기는 히라가나로, 한국어는 의미와 문화 설명에 사용하는 방향을 유지합니다.

## 알려진 제한 사항

- 실제 AI 연동, 생성 결과 저장과 생성 앱 미리보기 기능이 없습니다.
- 복사 실패 처리와 구형 브라우저용 대체 복사 방식이 없습니다.
- Tailwind CSS, Chart.js와 Google Fonts가 버전 고정 또는 로컬 보관 없이 외부 네트워크에 의존합니다.
- CDN 장애, 네트워크 차단, 콘텐츠 보안 정책 또는 제공자 변경 시 스타일·차트·글꼴이 정상적으로 표시되지 않을 수 있습니다.
- 접근성 자동 검사, 다국어 검사, 브라우저 호환성 테스트가 없습니다.
- 실제 배포 상태가 확인되지 않았습니다.

## 포트폴리오 전환 검토

최종 분류는 **C. 정적 전시 후보**입니다.

- 정적 교육 자료로서 수업 흐름, 문화 정보, 차트와 프롬프트 상호작용을 한 페이지에서 보여 주는 가치가 있습니다.
- 별도 서버, DB와 계정 운영이 필요 없어 정적 HTTPS 호스팅만으로 전시할 수 있고 운영 부담이 낮습니다.
- 전시에서는 현재 저장소가 AI 앱 자체가 아니라 수업 대시보드·프롬프트 빌더라는 점과, 프롬프트로 별도 생성할 목표 앱을 명확히 구분해야 합니다.

공개 전에는 다음 조치가 필요합니다.

- 담당자의 일본 문화·일본어 표현 감수
- Clipboard API 실패 처리와 대체 복사 방식
- 키보드 조작·스크린 리더 등 접근성 및 모바일·데스크톱 반응형 검증
- GitHub Pages를 포함한 실제 정적 HTTPS 배포와 링크 검증
- Tailwind CSS, Chart.js, Google Fonts CDN의 버전 고정 또는 자산 로컬화
- 소스·교육 콘텐츠·외부 자산·AI 생성 결과물의 사용 권리와 라이선스 확인

실제 AI 연동, 생성 결과 미리보기와 저장 기능을 구현하고, API 자격 증명을 클라이언트에 노출하지 않는 서버 측 구조와 안전한 입력·출력·개인정보·보존 정책까지 갖추면 **B. 웹앱 전환 후보**로 재분류할 수 있습니다.

## 라이선스와 외부 자산

저장소에는 라이선스 파일이나 사용 조건이 없습니다. 소스와 교육 콘텐츠의 재사용·배포 권한을 저장소 소유자가 명시해야 합니다.

또한 공개 전에 Tailwind CSS, Chart.js, Google Fonts의 라이선스와 CDN 이용 조건을 확인하고, 외부 AI가 생성한 문구·코드·이미지 등 추가 결과물의 사용 권리와 출처 표시 요구도 별도로 검토해야 합니다.
